WHAT THE EXISTING SYSTEM DOES
================================================================================

The autonomous-executor.py implements:

1. QUEUE MANAGEMENT
   - Canonical queue (tasks-queue.json)
   - Status tracking: pending -> assigned -> running -> in_progress -> completed
   - Load/save operations

2. AGENT ROUTING
   - Picks next task from queue
   - Routes to: Claude (implementation), Gemini (architecture), GPT (validation)
   - Assigns based on task keywords

3. LEARNING SYSTEM
   - Appends lessons to learning-history.jsonl
   - Retrieves recent learning
   - Prevents task duplication via recent_task_titles()

4. STEP TRACKING
   - Emits steps to agent-steps.jsonl
   - Records action, status, task_id, timestamp

5. SIGNAL DETECTION
   - site_signals(): Check home, robots.txt, sitemap, pages
   - token_signals(): Monitor Olist/Tiny token health
   - email_signals(): Check SMTP configuration

6. AUTO-GENERATION
   - Generates tasks when queue drops below watermark
   - Respects MAX_GENERATED_TASKS per cycle
   - Avoids duplicate task titles

7. INTERVENTION HANDLING
   - Reads from storage/private/agent-interventions.jsonl
   - Supports human-driven directives

8. FALLBACK SYSTEM
   - "Roo Auxiliars" - safe fallback helpers
   - QA/Self-test, Olist/Tiny, Frete/Checkout, General
   - Used when task execution isn't immediately safe

9. REPORT GENERATION
   - Cycle reports to autonomous-cycle-report.json
   - Event stream to autonomous-cycle-events.jsonl
   - Email health tracking
