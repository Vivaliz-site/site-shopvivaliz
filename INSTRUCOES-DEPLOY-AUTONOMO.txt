DEPLOY AUTONOMO SHOPVIVALIZ

1. OBJETIVO
Deixar o deploy funcionando sozinho para:
- fredmourao-ai/site-shopvivaliz
- fredmourao-ai/-shopvivaliz-pipeline

Depois da configuracao inicial:
- push em main
- GitHub chama o webhook
- servidor baixa e aplica o codigo automaticamente

2. ARQUIVOS IMPORTANTES
- deploy-webhook.php
- scripts/setup-auto-deploy.py
- scripts/setup-auto-deploy.ps1
- scripts/.ftp-credentials.example

3. PRIMEIRA CONFIGURACAO
Crie o arquivo local:
- scripts/.ftp-credentials

Use este modelo:

FTP_SERVER=ftp.shopvivaliz.com.br
FTP_USERNAME=seu_usuario_ftp
FTP_PASSWORD=sua_senha_ftp
FTP_PORT=21
FTP_REMOTE_DIR=/public_html/dev
GITHUB_TOKEN=seu_token_github

Observacoes:
- esse arquivo fica local e esta ignorado no Git
- o token do GitHub precisa ter escopos repo e admin:repo_hook

4. RODAR O BOOTSTRAP UMA VEZ
No PowerShell, dentro do projeto:

powershell -NoProfile -ExecutionPolicy Bypass -File scripts/setup-auto-deploy.ps1 -Bootstrap -Repos all

Isso faz:
- envia deploy-webhook.php para o servidor
- grava DEPLOY_SECRET no .env do servidor
- grava GITHUB_TOKEN no .env do servidor
- grava DEPLOY_REPOS no .env do servidor
- cria os webhooks dos dois repositorios no GitHub

5. RODAR SEM INTERACAO
Depois da primeira configuracao, rode:

powershell -NoProfile -ExecutionPolicy Bypass -File scripts/setup-auto-deploy.ps1 -Repos all

Se scripts/.ftp-credentials existir, ele entra em modo nao interativo automaticamente.

6. RODAR SO UM REPOSITORIO
Exemplos:

powershell -NoProfile -ExecutionPolicy Bypass -File scripts/setup-auto-deploy.ps1 -Repos site-shopvivaliz

powershell -NoProfile -ExecutionPolicy Bypass -File scripts/setup-auto-deploy.ps1 -Repos -shopvivaliz-pipeline

7. COMO FICA O FLUXO AUTOMATICO
Depois do bootstrap:
- voce faz commit
- voce faz push para main
- GitHub envia webhook
- o servidor valida assinatura
- o servidor identifica qual repositorio disparou
- o servidor baixa ZIP do repo correto
- o servidor extrai e aplica os arquivos

8. O QUE O WEBHOOK USA NO SERVIDOR
No .env do servidor ficam:
- DEPLOY_SECRET
- GITHUB_TOKEN
- DEPLOY_REPOS

9. TESTE RAPIDO
1. rode o bootstrap
2. confirme que o webhook foi criado no GitHub
3. faca um commit pequeno em main
4. faca push
5. verifique no servidor:
- logs/deploy-webhook.log

10. SE DER ERRO
Verifique:
- scripts/.ftp-credentials esta preenchido
- GITHUB_TOKEN tem permissao repo e admin:repo_hook
- FTP_REMOTE_DIR aponta para a pasta correta do site
- o servidor consegue escrever em logs/
- o arquivo deploy-webhook.php foi enviado

11. COMANDO MAIS IMPORTANTE
Para deixar tudo pronto sem perguntas:

powershell -NoProfile -ExecutionPolicy Bypass -File scripts/setup-auto-deploy.ps1 -Repos all
