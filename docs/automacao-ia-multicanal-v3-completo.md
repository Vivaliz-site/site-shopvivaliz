## 5. SETUP COMPLETO (15 PASSOS)

### FASE 1: PREPARAÇÃO (Passos 1-5)

#### Passo 1: Criar Conta Make.com
```
1. https://make.com
2. Sign up com email
3. Verificar email
4. Criar organização "ShopVivaliz"
5. Copiar token (Settings → API Keys)
```

#### Passo 2: Preparar Tiny/Bling ERP
```
TINY ERP:
1. Painel → Integrações → API
2. Criar token "Make.com Auto"
3. Copiar: TOKEN, API Base URL
4. Notas de acesso:
   - Host: https://api.tiny.com.br/v1/
   - Auth: Bearer {TOKEN}

BLING ERP:
1. Painel → Integrações → OAuth
2. Gerar CLIENT_ID e CLIENT_SECRET
3. Host: https://www.bling.com.br/Api/v3/
4. Salvar credentials em local seguro
```

#### Passo 3: Configurar OpenAI
```
1. https://platform.openai.com
2. Sign up / Login
3. Settings → API Keys → Create new
4. Copiar chave (formato: sk-proj-xxx)
5. Guardar em cofre (NÃO compartilhar)
6. Definir limite de uso: R$ 50/dia (começa baixo)
7. Testar: curl com hello world
```

#### Passo 4: Setup Cloudinary
```
1. https://cloudinary.com
2. Sign up (gratuito para testes)
3. Dashboard → Settings → Upload
4. Copiar: Cloud Name, API Key, API Secret
5. Criar pasta: "shopvivaliz-products"
6. Criar transformações:
   ├─ tiktok_shop_auto (800x1200)
   ├─ amazon_standard (1000x1000)
   ├─ mercado_livre_compact (600x600)
   └─ shopify_premium (1200x1200)
```

#### Passo 5: Criar Produto Piloto no ERP
```
EXEMPLO TESTE:
SKU: TEST-001
Nome: Rodízio 35mm Giratório com Freio
Categoria: Ferragens
Preço: R$ 25,90
Imagem: Qualidade 1024x1024px, fundo branco
Descrição: "Rodízio de qualidade para móveis..."
Estoque: 100 unidades

⚠️ IMPORTANTE: Use EXATAMENTE esses dados no Make.com
```

### FASE 2: INTEGRAÇÃO (Passos 6-10)

#### Passo 6: Criar Cenário Make.com
```
1. Make → Scenarios → Create new
2. Nome: "ShopVivaliz - Auto Multi-Canal"
3. Descrição: "Automação de produtos ERP → 4 canais"
4. Template: Start from scratch
5. Adicionar primeiro módulo: Tiny/Bling Trigger
6. Salvar (Ctrl+S)
```

#### Passo 7: Configurar Trigger (ERP)
```
TINY TRIGGER:
Module: Tiny ERP → Get Products
Auth: Usar token gerado no Passo 2
Request:
  GET /v1/produtos
  Params: {filtro: "alterado >= 1 day ago", limite: 10}
Polling: Every 2 hours

BLING TRIGGER:
Module: Bling ERP → List Products
Auth: OAuth (CLIENT_ID + SECRET)
Params: {filter: "modified >= 24h"}
Polling: Every 2 hours
```


```
A: Sistema de validação rejeita (Score < 70):
   1. IA tenta regenerar (1x apenas)
   2. Se ainda falhar: usa template genérico
   3. Admin recebe notificação para revisar
   4. Produto não publica até aprovação
   
   Resultado: 0% de produto ruim em produção
```

**P: TikTok/Amazon/ML APIs caem. Qual é o plano?**
```
A: Tratamento de erro automático:
   1. Retry com backoff exponencial (3x)
   2. Se 3x falham: marcar como "Pendente manual"
   3. Notificar admin
   4. Tentar novamente em 1 hora
   5. Escalar se > 4 horas down
   
   Resultado: Nenhum produto perdido
```
